1. "Goal"
Return ONLY valid JSON.

2. "Inputs"
- Log files from the last run
- Current run log files
- Archived broken log file (optional, if there is missing data)

3. "Steps"
1. Compare the current run log files with the last run log files.
2. Ignore the archived broken log file unless there is missing data in the current or previous run log files.
3. Ensure the summary is concise and suitable for email communication.
4. Include reasons for any failed operations in the summary.
5. Verify that the parser used the new config, taking into account recent changes to the configuration.
6. If timing changes significantly between runs, mention this in the summary.
7. Identify and report invalid rows and malformed JSON.

4. "Output"
A detailed report on the comparison of log files, including reasons for any failed operations, a concise summary suitable for email communication, and information about the parser's use of the new config.
